## Analysis

### Extension vs Web Portal

The web portal's **100% accuracy** significantly outperforms the extension's best agent (rule-based at 57.9%). Key differences:

1. **Type-aware filtering**: Web portal enforces field type constraints (email → email, phone → phone)
2. **Name heuristics**: Special-case handling for "customer name", "first name", "full name"
3. **Local embeddings**: Deterministic trigonometric hashing, no API required
4. **Better selectors**: Name attribute priority over autogenerated IDs

### Why Extension Agents Show 0ms Latency

The benchmark JSONs don't include duration fields — this is a data collection issue, not agent performance. Real latency would be:
- Rule-based: ~500-1000ms (regex + DOM manipulation)
- Embedding-matcher: ~1000-2000ms (embedding inference)
- MCP-agent: ~2000-5000ms (LLM API calls + tool invocation)

### Why MCP-agent Underperforms

Only 0.9% accuracy despite being LLM-powered suggests:
- Auth failures (likely same 403 issue blocking live tests)
- Empty responses from GitHub Models API
- No fallback to local strategies

### Why Embedding-matcher Shows 3.7%

Only 9 forms tested vs 26 for others — insufficient sample size. Also:
- No type-aware filtering (semantic matches put wrong values in wrong field types)
- No name-specific heuristics
- Missing the improvements now in web portal's smart-matcher
